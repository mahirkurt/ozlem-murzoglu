## Bolum 2: Duygusal Tasarim Teorisi

### 2.1 Don Norman'in Uc Katmani

Tum tasarim kararlari bu uc katmani ele almalidir:

#### Visceral (Icgudusel) Katman
Ilk izlenim - gorunum ve his.

```
Uygulamasi:
├── Renk paleti ilk bakista guven verici mi?
├── Tipografi profesyonel ve sicak mi?
├── Gorseller cocuk sagligi hissi veriyor mu?
└── Genel estetik modern ve temiz mi?
```

#### Behavioral (Davranissal) Katman
Kullanilabilirlik - urun isini ne kadar iyi yapiyor.

```
Uygulamasi:
├── Randevu alma sureci 3 tiktan az mi?
├── Bilgi hiyerarsisi acik mi?
├── Navigasyon sezgisel mi?
└── Formlar doldurulmasi kolay mi?
```

#### Reflective (Dusunsel) Katman
Anlam - tasarimi dusunme ve yorumlama.

```
Uygulamasi:
├── Kullanici "Bu klinige guvenebilirim" diyor mu?
├── Marka hatirlanabilir mi?
├── Deneyim paylasma istegi uyandiriyor mu?
└── Uzun vadeli sadakat olusturuyor mu?
```

### 2.2 Mikro-Duygular (Micro-Emotions)

Her etkilesim **mikro-duygular** dizisi olusturur - kullanici deneyimini sekillendiren ince, kisa sureli hisler.

#### Mikro-Duygu Haritasi

```
Kullanici Yolculugu                    Duygu Firsati
─────────────────                      ──────────────
Sayfa yuklenme         ───────────►    Merak, Beklenti
Ilk gorunum            ───────────►    Guven, Sicaklik
Buton hover            ───────────►    Kontrol, Yanit Alma
Klik animasyonu        ───────────►    Tatmin, Onay
Form tamamlama         ───────────►    Basari, Rahatlama
Hata mesaji            ───────────►    Anlayis, Destek (hayal kirikl. degil!)
```

### 2.3 Tasarim Kisitlari

Expressive tasarim denge gerektirir:

| YAPILMALI | YAPILMAMALI |
|-----------|-------------|
| Taninmis UI kaliplarini koru | Etiketleri kaldirma |
| Erisebilirlik standartlarini izle | Asinaligi boz |
| Islevselligi gorsel suslemeden once dusun | Animasyonu asiri kullan |
| Baglamsal uygunlugu degerlenldir | Her yerde "eglenceli" olma |

## Bolum 5: Algisal Renk Bilimi

### 5.1 OKLCH Renk Uzayi

Modern tasarimda **OKLCH** renk uzayi kullanilir. OKLCH, **algisal olarak esit** renk dagilimi saglar - farkli ton ve doygunluklarda bile tutarli parlaklik.

#### OKLCH vs Geleneksel Renk Uzaylari

| Ozellik | RGB/HEX | HSL | OKLCH |
|---------|---------|-----|-------|
| Algisal esitlik | Hayir | Hayir | Evet |
| Ton koruma | Kotu | Orta | Iyi |
| Kontrast tahmin | Zor | Zor | Kolay |
| Gamut esleme | N/A | N/A | Evet |
| CSS destegi | Tam | Tam | Modern |

#### OKLCH Formati

```
oklch(L% C H)

L = Lightness (Aydinlik)  : 0% (siyah) - 100% (beyaz)
C = Chroma (Doygunluk)    : 0 (gri) - ~0.4 (en doygun)
H = Hue (Ton)             : 0-360 derece

Ornek: oklch(57% 0.15 194)
       │      │     └── Ton: Teal/Camgobegi
       │      └── Doygunluk: Orta-yuksek
       └── Aydinlik: Orta
```

### 5.2 HCT ve OKLCH Karsilastirmasi

Google MD3 icin **HCT** (Hue, Chroma, Tone) kullanir; biz **OKLCH** tercih ediyoruz:

| Ozellik | HCT (Google) | OKLCH (Bizim) |
|---------|--------------|---------------|
| Temel | CAM16 + CIE Lab | OkLab |
| Ton koruma | Iyi | Cok iyi (ozellikle mavilerde) |
| Hesaplama | Karmasik | Basit |
| CSS desteği | Yok | Native (`oklch()`) |
| Algisal esitlik | Iyi | Cok iyi |

## Bolum 7: Surface Sistemi (Tonal Elevation)

### 7.1 Tonal Elevation Felsefesi

MD3 Expressive, **golge yerine ton farki** ile derinlik olusturur. Bu yaklasim:
- Daha modern gorunum saglar
- Dark mode'da daha iyi calisir
- Performans acisindan daha verimlidir
- Erisebilirlik icin daha uygun

### 7.2 Surface Container Hiyerarsisi

```
LIGHT MODE - Surface Hierarchy (En Acik → En Koyu)

     surface-container-lowest   ┃ oklch(100% 0 0)
              ▼                 ┃ En yuksek "yuzerlik" hissi
     surface-container-low      ┃ oklch(98% 0 0)
              ▼                 ┃
     surface                    ┃ oklch(98% 0.005 85)  ← Sicak tint
              ▼                 ┃ Temel yuzey
     surface-container          ┃ oklch(97% 0 0)
              ▼                 ┃ Varsayilan container
     surface-container-high     ┃ oklch(94% 0 0)
              ▼                 ┃
     surface-container-highest  ┃ oklch(92% 0 0)
                                ┃ En dusuk ama gorsel olarak "en yakin"
```

# KISIM V: HAREKET SISTEMI (MOTION)

## Bolum 13: Spring Physics (Yay Fizigi)

### 13.1 Neden Yay Fizigi?

Geleneksel easing curve'leri mekanik hisseder. **Spring physics** ise organik ve canli animasyonlar saglar.

```
Geleneksel Easing          Spring Physics
        │                        │
    ────┘                    ────╮
                                 │
   Hedef ─────────────      ────┤ (Overshoot)
                                 │
                             ────╯

   Dogusal, tahmin           Canli, organik
   edilebilir                beklenmedik ama
                             kontrolluu
```

### 13.2 Yay Fizigi Prensipleri

| Prensip | Aciklama |
|---------|----------|
| **Mass (Kutle)** | Ogeler fiziksel ağırlığa sahipmis gibi hisseder |
| **Stiffness (Sertlik)** | Yayin ne kadar hizli tepki verdigi |
| **Damping (Sonumleme)** | Salınımın ne kadar cabuk durdugu |
| **Overshoot (Asim)** | Hedefi gecip geri donme |

### 13.3 Easing Tokenlari

#### Standard Easing (Utility)

Basit UI degisiklikleri, fade'ler, renk gecisleri.

```scss
:root {
  --md-sys-motion-easing-standard:            cubic-bezier(0.2, 0, 0, 1);
  --md-sys-motion-easing-standard-decelerate: cubic-bezier(0, 0, 0, 1);
  --md-sys-motion-easing-standard-accelerate: cubic-bezier(0.3, 0, 1, 1);
}
```

#### Emphasized Easing (Important)

Kullanicinin dikkatini ceken gecisler.

```scss
:root {
  --md-sys-motion-easing-emphasized:            cubic-bezier(0.2, 0, 0, 1);
  --md-sys-motion-easing-emphasized-decelerate: cubic-bezier(0.05, 0.7, 0.1, 1);
  --md-sys-motion-easing-emphasized-accelerate: cubic-bezier(0.3, 0, 0.8, 0.15);
}
```

#### Expressive Easing (Hero Moments) - SPRING PHYSICS

En onemli anlar, buyuk layout degisiklikleri, modaller.

```scss
:root {
  // BOUNCE EASING - Hedefi asar ve geri doner
  --md-sys-motion-easing-expressive-standard:   cubic-bezier(0.4, 1.4, 0.2, 1);
  --md-sys-motion-easing-expressive-decelerate: cubic-bezier(0, 1.4, 0.2, 1);
  --md-sys-motion-easing-expressive-accelerate: cubic-bezier(0.4, 0, 0.2, 1);

  // Linear (ozel durumlar)
  --md-sys-motion-easing-linear: linear;
}
```

### 29.2 Kontrast Kontrolu

OKLCH'de kontrast tahmini kolay:

```
Lightness farki >= 40 birim genellikle 4.5:1 saglar

Ornek:
- on-surface:  oklch(24% 0 0)   = L: 24
- surface:     oklch(98% 0 0)   = L: 98
- Fark:        98 - 24 = 74     ✓ Yeterli
```
